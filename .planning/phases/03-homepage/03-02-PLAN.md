ICON MAPPING from `service.iconName` (strings on services.ts):
- `"Home"` → `Home`
- `"Layers"` → `Layers`
- `"Droplets"` → `Droplets`
- `"AlertTriangle"` → `AlertTriangle` (confirmed via research — Phase 3 research verified the alias exists in lucide-react v1.8.0)

Because Tailwind v4 CSS-first requires static class names (no dynamic string interpolation) AND because lucide-react icons are React components (not strings), build an explicit lookup map:

```tsx
const iconMap = {
  Home,
  Layers,
  Droplets,
  AlertTriangle,
} as const;
type IconKey = keyof typeof iconMap;
```

Then inside the map: `const Icon = iconMap[service.iconName as IconKey];` and render `<Icon className="h-6 w-6" aria-hidden="true" />`.

`service.description` is the long paragraph from services.ts. It is too long for a 4-card grid. Truncate in JSX with CSS: wrap the `<p>` in a className with `line-clamp-3` if available, OR simply slice the string: `{service.description.slice(0, 120)}…`. Prefer the string slice approach — it is deterministic and guaranteed to work without adding a plugin.

Example slice pattern for the `<p>`:
```tsx
<p className="mt-3 flex-1 font-body text-sm text-neutral-700">
  {service.description.length > 140
    ? `${service.description.slice(0, 140).trimEnd()}…`
    : service.description}
</p>
```

