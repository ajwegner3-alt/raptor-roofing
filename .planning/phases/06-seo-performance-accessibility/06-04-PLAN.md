**Fix 1 — TestimonialCarousel dot buttons (SEVERE, 12×12 → 48×48 hit area):**
File: `src/components/sections/TestimonialCarousel.tsx`

Current code (around line 87 per RESEARCH.md) is something like:
```jsx
<button onClick={...} className="h-3 w-3 rounded-full bg-...">...</button>
```

The visual dot SHOULD stay small (12×12 looks right in carousels), but the interactive hit area must be 48×48. Two acceptable patterns — pick whichever reads cleaner in context:

Pattern A (preferred — invisible hit area via padding):
```jsx
<button
  onClick={...}
  aria-label={`Go to testimonial ${i + 1}`}
  className="flex h-12 w-12 items-center justify-center rounded-full"
>
  <span className="h-3 w-3 rounded-full bg-... transition-colors" aria-hidden="true" />
</button>
```

Pattern B (pseudo-element hit area):
```jsx
<button
  onClick={...}
  aria-label={`Go to testimonial ${i + 1}`}
  className="relative h-3 w-3 rounded-full bg-... before:absolute before:inset-[-18px] before:content-['']"
/>
```

Pattern A is easier to reason about and more accessible — use Pattern A unless the existing layout (flex gap etc.) pushes you toward B. Either way, the <button> ELEMENT itself must be 48×48 minimum and must have an `aria-label` if it has no visible text.

**Fix 8 — neutral-500 contrast check (OPEN QUESTION from RESEARCH.md Q9):**
neutral-500 = `#4a5c62`. Page background = `#faf7f2`. Compute luminance + contrast ratio:
```

Ratio = (0.936 + 0.05) / (0.108 + 0.05) ≈ 6.24
```
