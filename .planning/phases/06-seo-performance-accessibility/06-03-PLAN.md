Commit real free-use roofing photos at every path the codebase already references, so Lighthouse can measure LCP against actual decoded pixels and the pitch demo doesn't show a broken-image icon. This plan is the CONTEXT.md deliverable "Commit real placeholder roofing photos in Phase 6" and closes the Pitfall 8 gap flagged in RESEARCH.md (missing files will 404, destroy LCP, and embarrass the pitch).

Purpose: Right now, 6 hero images + the default OG image are referenced in code but do not exist on disk. Next.js Image does not validate files at build time — the pages render but the browser gets 404s, LCP tanks, Lighthouse Best Practices penalizes the broken OG tag, and the demo shows broken-image icons. This plan sources the photos from Unsplash/Pexels (both licenses permit commercial use with no attribution required — RESEARCH.md Q6 verified), converts them to WebP <100 KB each, commits them, and documents every source in `SOURCES.md` so Phase 8 handoff knows exactly what to swap for real Raptor photos.

Output: 6 committed WebP files + 1 committed JPG + 1 SOURCES.md + minimal src updates (services.ts heroImagePath renames, Hero.tsx src path update). This plan contains a checkpoint task because photo sourcing requires human judgment on which specific photos look "Omaha/Midwest residential roofing" enough to not look obviously stocky.
</objective>
