**File 1: `src/components/layout/StickyMobileCTA.tsx`**

Can be a simple RSC (no state needed). No "use client" directive.

```tsx
import { Phone } from "lucide-react"
import { siteConfig } from "@/content/site"

export function StickyMobileCTA() {
  return (
    <div
      className="fixed inset-x-0 bottom-0 z-40 flex md:hidden bg-accent-600 text-white shadow-[var(--shadow-cta)]"
      style={{ paddingBottom: "env(safe-area-inset-bottom)" }}
      role="group"
      aria-label="Contact actions"
    >
      <a
        href={siteConfig.phone.href}
        className="flex flex-1 items-center justify-center gap-2 px-4 py-4 font-display text-base font-bold uppercase tracking-wide border-r border-white/20"
        aria-label={`Call ${siteConfig.name} at ${siteConfig.phone.display}`}
      >
        <Phone className="h-5 w-5" aria-hidden="true" />
        Call Now
      </a>
      <a
        href="/#estimate-form"
        className="flex flex-1 items-center justify-center gap-2 px-4 py-4 font-display text-base font-bold uppercase tracking-wide"
      >
        Free Estimate
      </a>
    </div>
  )
}
```

Notes:
- `fixed inset-x-0 bottom-0` — pinned to viewport bottom
- `md:hidden` — hides on tablet/desktop (768px+)
- `z-40` — below sticky header (z-50) and mobile overlay (z-100), above content
- `env(safe-area-inset-bottom)` via inline style — iPhone notch safe area. NOT a Tailwind arbitrary value because we want direct env() support
- Two equal-weight buttons via `flex-1`
- Py-4 + text-base yields ~56px button height — comfortably > 48px tap target
- `/#estimate-form` hardcoded href per CONTEXT resolution — works on homepage as anchor, 404s gracefully on other pages until Phase 4/5 wires /contact (acceptable placeholder behavior for Phase 2 standalone deploy; homepage is the only page that exists in Phase 2 anyway)
- Named export
